# 模拟评论覆盖补齐修复记录

时间：2026-04-28 16:59:18

## 结果

已修复“模拟评论区部分评论没有跟随当前主题更新”的问题。根因不是豆包认为不会出现这些评论，而是服务端在豆包返回评论不足时，用旧的本地 mock 评论补齐，导致洛克王国世界案例混入宿舍、收纳、学生等跨主题评论。

## 改动

- `server/lib/preflight/prompt.ts`
  - quick / deep 模式都要求豆包固定输出 8 条评论。
  - 明确要求 2 条 core、2 条 broad、2 条 weak、2 条 misfire。
  - 明确不能因为觉得弱相关或误推评论概率低就省略。
- `server/lib/preflight/runPreflightSimulation.ts`
  - 不再使用旧 mock 评论补齐真实 provider 的 `simulatedReplies`。
  - 新增当前主题感知的覆盖补齐逻辑。
  - 如果豆包仍然漏掉某类人群，系统会按当前标题、正文、脚本和安全边界生成同主题补齐评论。
  - 补齐评论会标注为 `补齐模拟`，避免被误认为豆包真实返回。
- `src/pages/PreflightStudioPage.tsx`
  - 评论区标题在存在补齐项时显示“x 条覆盖补齐”。
  - 显示提示：豆包未返回全部人群评论，补齐项需人工复核。
- `src/styles/pages.css`
  - 覆盖补齐评论使用虚线和提示色，和豆包原始评论区分。
- `server/lib/preflight/runPreflightSimulation.test.ts`
  - 新增断言：补齐评论必须同主题，不能再混入宿舍、收纳、桌面等旧语境。
