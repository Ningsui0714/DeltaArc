# Preflight 输出结构 V2 实施记录

## 结果

已按设计方案把 Preflight 输出从 V1 粗粒度报告升级为 V2 细粒度结构，并完成真实 provider 双案例 Browser Use 验证。

## 改动范围

- `shared/preflightSimulation.ts`
  - 新增 `schemaVersion="preflight_v2"`、`scenario`、`contentPromise`、`visualRead`、`audienceDistribution`、`riskReview`、`simulatedMetrics`、`qualityCheck`。
  - 为 `simulatedReplies` 新增 `comment`、`surfaceIntent`、`hiddenNeed`、`suggestedReply`、`contentAction`、`riskSignal`、`evidenceNeeded`。
  - 为 `interventions` 新增 `problem`、`change`、`exampleRewrite`、`expectedEffect`、`effort`、`evidenceSource`。
  - 保留 V1 字段，并提供 V1 -> V2 自动适配，旧 mock 和旧远端结果仍可解析。

- `server/lib/preflight/prompt.ts`
  - Prompt contract 切到 V2 strict JSON。
  - 强制区分 `suggestedReply` 和 `contentAction`。
  - 强制每条 intervention 输出 `problem/change/exampleRewrite/expectedEffect/effort/evidenceSource`。
  - 增加场景隔离，校园提交场景禁止套用宿舍/KOC 话术。

- `server/lib/preflight/doubaoProvider.ts`
  - quick `max_output_tokens` 从 4200 提到 7200。
  - deep `max_output_tokens` 从 7200 提到 9000。
  - 保留 `thinking.disabled` 和 `json_object`，避免 reasoning 超时和非 JSON 输出。

- `src/pages/PreflightStudioPage.tsx`
  - 右侧控制台展示模拟值免责声明、真实推荐回复、内容动作、补证据、示例改写和质量检查。
  - 校园案例切换为 `campus_ai_competition + submission_readiness`，并按场景切换首屏标题、作者、平台和输入占位文案。

- `src/styles/pages.css`
  - 增加 V2 指标免责声明、评论内容动作和示例改写样式。

- `.env.example` / `.env.local`
  - `DOUBAO_TIMEOUT_MS` 调整为 150000，适配 V2 大 JSON 真实 provider 响应。

- `harness/run-generated-preflight-cases.mjs`
  - 轮询超时调整为 180000。
  - 摘要增加 `schemaVersion/scenario/suggestedReply/contentAction/exampleRewrite/qualityScore`。

## 风险与回滚

- V2 结构字段更多，真实 provider 单次耗时约 100-120 秒；已通过 150 秒超时和 7200 token quick 上限兜住。
- 如果真实 provider 成本或耗时不可接受，可临时将 `.env.local` 的 `PREFLIGHT_PROVIDER=mock` 或降低输出条目数。
- V1 字段仍保留，回滚前端展示时不需要迁移历史结果。
