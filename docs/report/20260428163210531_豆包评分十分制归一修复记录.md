# 豆包评分十分制归一修复记录

时间：2026-04-28 16:32:10

## 结果

已修复豆包把 `7/10` 口径输出成数字 `7` 后被前端显示为 `7/100` 的问题。现在评分字段会兼容三种口径：

- `0.7` -> `70/100`
- `7` -> `70/100`
- `70` -> `70/100`

流量占比字段不会被误放大，例如 `exposureShare=7` 仍表示 `7%`，不会变成 `70%`。

## 解释

这次低到 `7/100` 不是内容真的只值 7 分，而是远端模型输出口径偏成了十分制。模型判断大概是“7 分/10 分制”，但页面按百分制展示，导致观感异常。修复后它会显示为更合理的 `70/100`。

内容本身仍会被扣分，主要原因是：

- `99 元`预算范围需要核价。
- 推荐单品是否合规需要复核。
- 具体单品细节不足。
- 关注转化设计还不够明确。

所以合理结果不是满分，而是中等偏可发布前修改的分数。
