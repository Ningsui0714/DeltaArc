## 我的感受

这次改完后，样例终于更像“发小红书前先试一下”的产品，而不是通用分析器。咖啡盲测和双人开黑这两个最自然，封面、标题、评论动作和用户动机都比较顺，像真实用户会点开的内容。

异环和原神两个样例更偏运营预演，视觉冲击够强，但边界也更重：AI 图只能做内部预演，正式发布必须换官方授权素材；原神样例还必须按发布时间重新核对活动状态，尤其是 2026-04-28 之后不能把 2026-04-07 已结束的活动写成进行中。

页面体验上，我觉得核心闭环已经成立：左侧是待发布稿，中间像手机发布预览，右侧给评论、动作、质量和风险。短板是 mock 评论仍然偏通用，真实 provider 或 deep 模式才能更像每个题材的真实评论区。

## 风险清单

- AI 图不能直接替代商业发布素材，尤其是命名 IP 或官方运营内容。
- 游戏类样例需要避免暗示官方合作、未公开信息、福利承诺和抽卡收益。
- 小红书样例不能诱导刷量，评论引导要像自然互动。
- mock provider 适合页面回归，不代表真实模型输出质量。
- Browser Use 全页截图在当前窄视口下会重复部分页面块，报告采用视口截图和 DOM 信号作为验收依据。
