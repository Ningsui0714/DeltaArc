## 案例 1：宿舍桌面改造

Browser Use 截图：

```text
harness/output/browseruse-ark-real-dorm-normalized.png
```

质量评分：`7.5/10`

优点：

- 读图准确：识别到“99元改造”“前后对比”“宿舍桌面”三个核心信号。
- 风险抓得对：指出“没有标注单品价格，看不清具体新增了哪些物品”。
- 评论有真实感：包含“求清单”“99元是不是全套桌子”“标题党”等合理评论。
- 动作可执行：能给出标题改法、预算拆分、置顶评论、价格标注、踩坑补充。
- 人群分层正常：`30% / 45% / 15% / 10%`，不再出现真实模型返回小数导致的 `0%` 问题。

问题：

- 部分理由重复，例如多条动作原因都接近“首屏信息决定停留”。
- 互动数字仍像模拟值，不能被用户误解成真实预测。
- 个别句子不够顺，例如“减少误解退出发”这类轻微病句仍可能出现。

## 案例 2：校园 AI 产品 Demo

Browser Use 截图：

```text
harness/output/browseruse-ark-real-campus-quality-fixed.png
```

质量评分：`7/10`

优点：

- 读图准确：识别到 DeltaArc、提交前试映、评审追问、风险预警、提交清单等关键信号。
- 风险有效：抓到“第三方工具，非大赛官方”的身份澄清问题。
- 建议可落地：提出放大核心功能区、标注第三方身份、改标题突出“怕漏材料/评审提问”、补工具示例。
- 评论基本合理：参赛队长、产品导师、备赛学生、桌面爱好者、求职学生等分层成立。

已修复问题：

- 原先右侧“推荐回复”使用前端硬编码宿舍模板，导致校园 Demo 输出出现“清单/桌面痛点/改造方案”串台。
- 已改为展示模型返回的 `reply.intervention`，复测后无宿舍模板泄漏。

剩余问题：

- 推荐回复仍偏短，例如“立即回复告知使用方式”，更像动作提示，不像可直接复制的回复。
- 弱相关/误推评论中仍有“看起来有点意思，但还需要更多信息”这种泛化句。
- 目前页面还是把它包装成 KOC Growth Lab 的增长控制台，校园 Demo 场景会稍微不贴合比赛评审语境。

## 下一步质量优化

1. 把 `reply.intervention` 改成更明确的双字段：`suggestedReply` 和 `contentAction`，避免“推荐回复”只显示动作提示。
2. Prompt 增加质量约束：禁止泛化句、禁止重复 action reason、每条建议必须包含“改哪里 + 怎么改 + 为什么”。
3. 页面增加“模拟值”标记，明确点赞/收藏/评论/转发不是预测承诺。
4. 对校园 Demo 使用独立评审视角页面，不再复用涨粉控制台文案。
