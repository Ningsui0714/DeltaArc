# 发布安全审查能力接入记录

时间：2026-04-28 14:58:17

## 结果

已把“大厂运营发布前是否会暴雷”的安全审查能力接入 Preflight 主链路。新结果字段为 `publishSafetyReview`，用于给出发布闸门、风险红旗、复核 checklist、升级对象和发布前必须修复项。

## 改动

1. 共享 schema

- 在 `shared/preflightSimulation.ts` 新增 `PreflightPublishSafetyReview`。
- 支持 `gate = go / revise / hold`：
  - `go`：未发现明确阻断项。
  - `revise`：先改再发。
  - `hold`：暂停发布并升级复核。
- 新增红旗维度：过度承诺、隐私授权、版权素材、平台规则、品牌声誉、舆情外溢、数据安全、未成年人保护、赛事边界、语境误读。
- 旧结果没有 `publishSafetyReview` 时，会基于 `risks / riskReview / qualityCheck` 自动生成保守兜底结论。

2. 模型 prompt

- 在 `server/lib/preflight/prompt.ts` 的 JSON contract 中加入 `publishSafetyReview`。
- 要求模型以“大厂发布安全审查”的视角检查：
  - 过度承诺和虚假功效。
  - 隐私、授权、版权和数据安全。
  - 平台规则、诱导互动和虚假营销。
  - 品牌、公关和舆情外溢。
  - 官方背书误读和 PCG 赛事边界。
  - 未成年人或校园场景敏感信息。

3. mock 和归一化

- 在 `server/lib/preflight/mockProvider.ts` 补充三类 mock 安全审查：
  - PCG 参赛提交：官方评审边界、样例数据脱敏、AI 能力边界。
  - KOC 增长发布：价格口径、商单误读、平台互动规则。
  - 通用品牌发布：品牌声誉、素材授权、舆情外溢。
- 在 `server/lib/preflight/runPreflightSimulation.ts` 中，当真实 provider 未给安全审查时，自动使用 mock 兜底，避免 UI 空缺。

4. 前端展示

- 在 `src/pages/PreflightStudioPage.tsx` 的右侧控制台新增“发布安全审查 / 提交安全审查”模块。
- 展示发布闸门、分数、升级复核对象、风险红旗、检查项、必须修复项。
- 在 `src/styles/pages.css` 补充安全审查卡片、红旗、检查项和移动端布局样式。

## 风险与边界

- 这不是官方法务结论，也不是平台内容安全最终判定；它是发布前预审工具，用于提前暴露需要人工复核的风险点。
- `hold` 只代表产品建议暂停发布并升级复核，不代表内容一定违法或违规。
- 真实 PCG 参赛提交前，仍需要人工检查素材授权、隐私脱敏、赛事材料边界和对外表达。

## 后续建议

1. 增加“安全审查专用输入”：是否官方账号、是否含真实人物、是否商业合作、是否未成年人、素材来源。
2. 给安全审查结果增加导出：一页“发布前复核单”，方便运营、品牌、公关、法务协作。
3. 如果要更像大厂流程，可以加入责任人、审批状态和复核记录。
