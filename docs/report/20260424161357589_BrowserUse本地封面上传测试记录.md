# Browser Use 本地封面上传测试记录

时间：2026-04-24 16:13:57

## 结果

已使用 Browser Use 插件在 in-app browser 中测试 `KOC Growth Lab` 的本地封面上传流程。页面加载正常，上传控件可见，控制台无错误；但 Browser Use 当前暴露的 Playwright locator 不支持 `setInputFiles`，也无法向系统文件选择框输入本地路径，因此无法通过插件把本地图片真正塞进 `<input type="file">`。

## 测试文件

- `D:\pcg-AIproduct\桌面改造示例图片.png`
- 图片内容：`99元宿舍桌面改造`，左侧改造前杂乱，右侧改造后整洁温馨。

## Browser Use 实测

- 打开 `http://127.0.0.1:5001/`：成功。
- 页面标题：`KOC Growth Lab`。
- 上传区域：`上传封面 / 作品截图` 可见。
- file input 数量：1。
- 直接上传尝试：失败，插件 locator 没有 `setInputFiles`。
- 对 `input[type="file"]` 执行 `fill(path)`：失败，浏览器拒绝填充 file input。
- 点击上传区域打开原生文件选择框：点击成功。
- 通过 Browser Use 键盘输入本地路径：失败，插件提示没有页面内可编辑元素聚焦。
- 浏览器控制台错误/警告：0 条。

## 判断

这是 Browser Use 插件能力限制，不是应用前端上传控件本身的问题。要完成真正的 UI 文件上传测试，需要用户手动在系统文件选择框里选图，或使用支持 `setInputFiles` 的浏览器自动化环境。

## 后续建议

- 若只验证真实多模态能力，可继续用本地 API 传 base64 图片。
- 若要验证完整 UI 上传链路，请手动选择该图片后让我继续用 Browser Use 点击推演并检查结果。
