# Preflight 风险修复实现记录

时间：2026-04-27 18:09:48

## 结果

本轮完成发布前预演链路的三项高优先修复：前端旧轮询防回写、示例数据与真实推演结果区分、远端失败显式 degraded/fallback 语义。

## 改动

- `src/hooks/usePreflightSimulation.ts`
  - 增加 active run id guard。
  - `reset()` 会让旧轮询失效，旧请求返回后不会再写入 job/result/status。
  - 前端状态新增 `degraded`。

- `shared/preflightSimulation.ts`
  - `PreflightJobStatus` 新增 `degraded`。
  - `PreflightSimulationResult` 新增 `degraded` 与 `fallbackReason`。
  - `PreflightSimulationJob` 新增 `fallbackReason`。
  - parser 保留 degraded/fallback 元数据。

- `server/lib/preflight/runPreflightSimulation.ts`
  - 远端 Doubao/火山方舟失败时，本地 mock fallback 会标记 `degraded: true` 并写入 `fallbackReason`。

- `server/lib/preflight/preflightJobStore.ts`
  - `completePreflightJob` 根据结果标记 `completed` 或 `degraded`。
  - degraded 任务显示“兜底完成”和明确兜底消息。

- `src/pages/PreflightStudioPage.tsx`
  - 未开始推演时，评论区显示“示例评论区”，指标为 `0 / 待生成`。
  - 右侧控制台未生成前只显示“推演完成后显示真实动作”，不再展示示例涨粉动作冒充结果。
  - degraded 结果在进度卡和控制台显示“本地兜底 / 兜底结果”。

- 测试
  - `shared/preflightSimulation.test.ts` 覆盖 degraded job/result 解析。
  - `server/lib/preflight/runPreflightSimulation.test.ts` 覆盖远端失败 degraded fallback。
  - `src/pages/PreflightStudioPage.test.tsx` 覆盖示例态文案。

## 风险与回滚

- 风险：`degraded` 是新增 job 状态，若外部调用方只识别 `completed/error`，需要同步适配。
- 回滚：可先保留 result 的 `degraded/fallbackReason`，把 job 状态临时回退为 `completed`；前端 hook 和示例态改动可独立回滚。
