# 发布前社媒预演器设计方案

时间：2026-04-26 22:05:53

## 一句话定位

上传一条待发布作品，提前看见它在小红书式信息流里的封面、正文、互动数据、评论区和涨粉机会。

## 设计判断

当前报告式推演更像 AI 分析工具，评委需要读完结果才理解产品价值。新版应该把主视觉改成“发布后的样子”：用户上传作品后，页面直接生成一张小红书风格帖子预览，下面出现模拟点赞、收藏、评论、转发和评论回复，右侧再给出 AI 增长诊断。

## 核心体验

1. 用户上传封面、标题、正文。
2. 系统生成一张手机帖子预览。
3. 预览页展示模拟互动数据和评论区。
4. AI 标出哪些评论代表涨粉机会、哪些评论是跑偏风险。
5. 用户可点击优化标题、优化正文、推荐回复，看到互动数据预测变化。

## 页面结构

### 顶部工作栏

- 产品名：`KOC Growth Lab`
- 当前任务：`发布前预演`
- 状态胶囊：`已读取图片`、`正在模拟评论`、`可优化`
- 主按钮：`生成预演`
- 次按钮：`一键载入示例`

顶部不做大 hero，避免像官网。它应像一个创作者工作台。

### 左侧：作品输入区

窄栏，宽度约 320px。

- 封面上传
- 标题输入
- 正文输入
- 人设/账号定位
- 目标动作
- 合规边界

输入区要收敛，不抢主视觉。默认可折叠成 `素材` 标签页。

### 中间：手机发布预览

这是第一视觉重点，占页面 40%-45% 宽度。

组成：

- 手机框，不要太拟物，做成干净的社媒预览容器。
- 顶部作者区：头像、昵称、关注按钮。
- 封面图：展示用户上传图片。
- 标题与正文：保持小红书笔记排版。
- 标签：如 `#宿舍改造`、`#低预算收纳`、`#学习桌面`。
- 互动条：点赞、收藏、评论、转发。
- 数据随推演状态逐步跳动，例如：
  - 点赞 `128 -> 356`
  - 收藏 `42 -> 190`
  - 评论 `18 -> 64`
  - 转发 `9 -> 27`

视觉重点不是“真实复刻小红书”，而是让用户一眼理解：这是发布前模拟。

### 中下：模拟评论区

评论区直接接在手机预览下方，或者作为手机预览的下半屏。

评论分五类：

- 核心粉：认真问清单、预算、踩坑。
- 泛兴趣用户：觉得好看、想收藏。
- 质疑用户：质疑价格真实性、效果夸大。
- 路过用户：轻互动、不一定关注。
- 潜在关注者：被系列感吸引，可能关注下一期。

每条评论包含：

- 用户头像与昵称
- 评论内容
- 用户类型标签
- 关注概率：高 / 中 / 低
- 推荐回复按钮

点击评论后，右侧面板显示：

- 为什么这条评论重要
- 推荐回复
- 是否值得置顶
- 是否能转化成下一篇内容选题

### 右侧：AI 增长控制台

宽度约 360px。

模块：

1. 内容读片
   - AI 从图片里看到什么。
   - 首屏误读风险。
   - 封面最强吸引点。

2. 互动预测
   - 首轮点击率
   - 收藏动机
   - 评论爆点
   - 关注转化机会

3. 跑偏风险
   - 价格质疑
   - 种草真实性
   - 只收藏不关注
   - 改造效果被误解

4. 优化按钮
   - `优化标题`
   - `补清单明细`
   - `生成置顶评论`
   - `生成评论回复`

点击优化后，不要只生成文字，要同步更新中间预览里的数据预测。

## 关键交互

### 一键载入示例

评委打开后可以点 `一键载入示例`，自动填入“99元宿舍桌面改造”的标题、正文和封面说明。这样不用上传也能看完整体验。

### 生成预演

点击后分阶段出现：

1. `读取封面`
2. `模拟首轮分发`
3. `生成用户评论`
4. `识别涨粉机会`
5. `生成优化动作`

阶段不是纯进度条，要让中间手机预览逐步长出来。

### 评论回复

点击某条评论的 `推荐回复`：

- 右侧生成 2-3 个回复选项。
- 标明回复目标：解释价格 / 引导收藏 / 引导关注 / 防止争议。
- 可把选中的回复插入到手机预览评论区。

### 优化前后对比

点击 `优化标题` 后显示：

- 原标题
- AI 建议标题
- 点赞、收藏、关注预测变化

例如：

- 点赞：`356 -> 430`
- 收藏：`190 -> 260`
- 关注：`31 -> 48`

## 视觉方向

关键词：真实、轻社媒、创作者工作台、发布前沙盘。

建议风格：

- 背景：浅灰蓝或淡纸白，避免大面积紫色渐变。
- 中间手机预览：白底、细边框、轻阴影。
- 强调色：小红书红作为动作色，但不要整页红。
- 数据色：蓝绿色用于增长变化。
- 风险色：暖黄或珊瑚色。
- 字体：中文以清晰现代为主，标题略有杂志感，但控制密度。

## 信息架构优先级

第一优先级：用户看到“作品发布后的样子”。

第二优先级：用户看到“评论和互动会怎么发生”。

第三优先级：用户知道“我应该怎么改、怎么回复、怎么涨粉”。

报告型内容放到右侧，不再作为主视觉。

## 参赛表达

这版界面可以这样讲：

`KOC Growth Lab 是一个发布前社媒增长沙盘。创作者上传待发布作品后，豆包多模态模型先读懂封面和正文，再模拟平台分发、用户评论、互动数据和涨粉机会，让创作者在真正发布前就能看到内容可能如何发展，并提前优化标题、正文和评论回复。`

## MVP 改造范围

第一版只做：

- 作品输入区
- 手机帖子预览
- 模拟互动数据
- 模拟评论区
- 右侧 AI 诊断
- 一键载入示例

暂不做：

- 多平台切换
- 真实账号授权
- 历史作品库
- 团队协作
- 发布到真实平台

## 验收标准

- 评委打开首页 5 秒内能理解产品是“发布前预演器”。
- 上传图片后，中间必须出现手机帖子预览。
- 生成结果必须包含点赞、收藏、评论、转发模拟数据。
- 生成结果必须包含不少于 6 条模拟评论。
- 每条评论必须标注用户类型或增长信号。
- 至少一条评论可生成推荐回复。
- 右侧必须保留 AI 读片、风险、优化动作。
- 一键示例可完整跑通，不依赖用户自己准备素材。
